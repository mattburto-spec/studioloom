# Design Lesson Timing: Quick Reference Card
## Print This. Use This. (March 2026)

---

## The 4-Phase Workshop Model (EVERY LESSON)

```
OPENING (5-10 min)          MINI-LESSON (8-15 min)       WORK TIME (30-60+ min)        DEBRIEF (5-10 min)
├─ Hook                     ├─ Teach ONE skill           ├─ Making/practice             ├─ Structured feedback
├─ Challenge                ├─ Demonstrate               ├─ Teacher circulates          ├─ Peer share
└─ Context                  └─ Check for understanding   └─ 1-on-1 conferences         └─ Bridge to next
```

**Rule:** Mini-lesson max = 1 + student age (Year 7 = max 13 min, Year 9 = max 14 min, Year 11 = max 16 min)
**Rule:** Work time must be ≥ 45% of period, ideally 60%+
**Rule:** Always have a debrief; never skip

## Period-Specific Timing Templates

### 45-Minute Period (TIGHT)
```
5 min:    Opening
10 min:   Mini-lesson (HARD LIMIT)
25 min:   Work time
5 min:    Debrief
────────
45 min    Total
```
⚠️ Feels rushed for design work; consider combining 2 periods for making time.

### 60-Minute Period (STANDARD)
```
5 min:    Opening
12 min:   Mini-lesson
38 min:   Work time (main event)
5 min:    Debrief
────────
60 min    Total
```
✓ Good for research, analysis, concept development. Teacher has time for conferences.

### 90-Minute Block (IDEAL)
**Option A (Instruction-Heavy):**
```
5 min:    Opening
25 min:   Mini-lesson 1
25 min:   Work time (guided practice)
5 min:    Refocus checkpoint
25 min:   Work time (more independent)
5 min:    Debrief
────────
90 min    Total
```

**Option B (Hands-On-Heavy):**
```
5 min:    Opening
12 min:   Mini-lesson
65 min:   Work time (with 30-min & 60-min check-ins)
8 min:    Debrief
────────
90 min    Total
```
✓ Best for prototyping, iterative making, sustained flow.

## Design Unit Phase Timing (6-8 Week Cycle)

| Week | Phase | Cumulative | Milestone |
|------|-------|-----------|-----------|
| 1-2  | Inquiry & Analysis | 4 periods | 5+ data points; design brief done |
| 2-3  | Research & Synthesis | 7-8 periods | Empathy maps, affinity diagrams |
| 3-4  | Ideation | 10-11 periods | 20+ ideas → 3-5 viable concepts |
| 4-5  | Concept Development | 13-14 periods | Detailed design, tech specs ready |
| 5-6  | Prototyping | 16-17 periods | Functional first prototype exists |
| 6-7  | Testing & Iteration | 19-20 periods | User feedback documented |
| 7-8  | Finalization & Exhibition | 22-24 periods | Portfolio, presentation, showcase |

**Early finisher check-ins:**
- End Week 1: 5+ data points gathered? → Yes = ready; No = extend research
- End Week 3: 3-5 concepts selected? → Yes = ready; No = more SCAMPER
- End Week 5: Functional prototype exists? → Yes = test; No = more making time

## Cognitive Load Rules (NO EXCEPTIONS)

### Work-Rest Cycles (Pomodoro-Informed)
```
25-30 min work → 5 min break (desk/analysis work)
30-40 min work → 5-10 min break (design/problem-solving)
45-60 min work → 10-15 min break (hands-on/making)
```

### Passive Phase Max Duration
```
READING/WRITING only:    max 20 min (Y7-9)   or max 25-30 min (Y10+)
                         → then insert 5-10 min ACTIVE phase

VIDEO only:             max 15 min (all ages)
                        → pause for discussion every 5-7 min

GROUP DISCUSSION only:   max 20-30 min (loses focus after this)
```

### Direct Instruction Max
```
30 minutes = the absolute ceiling before adolescent engagement drops
→ Split longer lessons into 2×15 min chunks with 5 min active break
```

## Extensions for Early Finishers (Pick by Phase)

### If They Finish Investigation (Week 2) Early
✓ Interview 2 more stakeholders (30 min)
✓ Analyze 3 competitor products (25 min)
✓ Accessibility audit of existing products (25 min)
✓ Secondary stakeholder empathy map (20 min)
✓ Trend research in their domain (20-30 min)

### If They Finish Ideation (Week 4) Early
✓ SCAMPER on their top concept (20-25 min)
✓ Rapid prototyping of 3 versions in foam/paper (30 min)
✓ Constraint-based variation (budget OR weight limits) (20 min)
✓ Technical feasibility research (25 min)
✓ Peer feedback session on top 3 ideas (15 min)

### If They Finish Prototyping (Week 6) Early
✓ Build second prototype with alternative materials (30-45 min)
✓ Scale-up or scale-down version (25 min)
✓ User testing with different demographic (20 min)
✓ Manufacturing/production method research (20 min)

### If They Finish Testing (Week 7) Early
✓ Edge-case testing (stress test, extreme use) (20 min)
✓ Long-term durability study (plan for next week) (15 min)
✓ Cross-cohort user comparison (15-20 min)
✓ Sustainability impact analysis (20 min)

## The Structured Debrief (Pick One)

### 5-10 Minute Whole-Class Debrief
```
1 min:  "Who wants to share?"
3-4 min: 2-3 students share (1-2 min each)
2 min:  Teacher synthesis: "I noticed..."
1 min:  Bridge: "Next time we'll..."
```

### 15-Minute Pair Feedback (Structured)
```
3 min:  Presenter explains what they were trying to do + main challenge
3 min:  Listener asks clarifying questions (no suggestions yet)
6 min:  Feedback rounds:
        • "I like..." (2 min: what worked)
        • "I wish..." (2 min: what could improve)
        • "I wonder..." (2 min: what they're curious about)
2-3 min: Presenter response: "I'm taking away..."
```

### 45-60 Minute Design Critique (Big Moment)
```
3 min:    Presenter frames: Who's the user? What's the goal? What's the constraint? What do I need?
5 min:    Silent observation (viewers examine work, jot thoughts)
10 min:   Clarifying questions (Understanding, not judgment)
15 min:   Feedback (Stickers/markers: "what works," "rethink this")
10 min:   Analysis (move through flagged items, deeper discussion)
5 min:    Note capture (what are the key insights?)
2 min:    Wrap-up (presenter summarizes takeaways)
```

## Common Pacing Mistakes & Fixes

| Mistake | Fix |
|---------|-----|
| **Mini-lesson > 20 min** | Cut to 10-15 min; teach rest "just-in-time" during work time |
| **Work time < 25 min** | Extend to minimum 30 min; combine 2 periods if needed |
| **No debrief** | Add 5-min reflection MINIMUM (even exit ticket counts) |
| **Open debrief that runs over** | Use STRUCTURED protocol with a TIMER |
| **One-size timing** | Build extension menu (for fast finishers) + flexible check-ins (for slow) |
| **Unmapped transitions** | Pre-announce: "At 2:45 we transition." Use a timer. |
| **Week 5 testing too short** | Allocate 2 full periods; leave Week 6 open for iteration |
| **Exhibition last-minute** | Start documentation in Week 4; exhibition practice in Week 7 |

## Cognitive Load Checklist

Before you finish lesson planning, ask:
- [ ] Mini-lesson ≤ (1 + age) minutes?
- [ ] Work time ≥ 45% of period?
- [ ] Any passive phase > 20 min (Y7-9) or > 30 min (Y10+)?
- [ ] If yes: insert 5-10 min break/active phase in the middle?
- [ ] Debrief included & structured (not open-ended)?
- [ ] Transitions pre-announced & timed?
- [ ] Extension activities ready for early finishers?
- [ ] Check-in points identified (for long work blocks)?

---

## If You Only Remember 3 Things

1. **Opening (5) + Mini-Lesson (12) + Work Time (38) + Debrief (5) = 60 minutes**

2. **Never do > 30 min direct instruction. Never do < 30 min work time.**

3. **Always have an extension menu ready. Early finishers don't sit idle.**
