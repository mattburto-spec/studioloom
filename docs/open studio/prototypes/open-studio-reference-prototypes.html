Loominary · reference prototypes

Eight patterns worth borrowing

Each prototype demonstrates one mechanism the Open Studio mode and student dashboard could borrow from. None of them rely on a chatbot.

design constraint · v1
No student-facing chatbot at launch. Every "help" surface must be a curated, human-authored, navigable artefact — not generated text. Hints, next steps, and skill resources are linked, not synthesised.

01
borrowed from · scrum daily standup

Three questions, asynchronous

Did, will do, blocking. The whole ritual in 60 seconds.

Lin's standup · tue · 09:14
Yesterday
Got the breadboard wired but the LED stayed dark.
Today
Trace the circuit with a multimeter, find the break.
Blocker
Not sure how to use the multimeter properly.
try one of these →
→ skill card · multimeter basics → video · 90s continuity test → ask peer · Maya, Eli
what it does
Three questions, no more. Closes the loop on yesterday and opens today's intent. The blocker is the only one that gets follow-up.
how it applies, no chatbot
The blocker text is matched against a tagged skill library. The student sees three curated options — a skill card, a short video, and named peers — none of it generated. Teacher-authored, taggable, auditable.

Body doubling & commitment

what it does

borrowed from · notion + roam daily notes

Templated daily reflection

A page per day with the same prompts. Friction goes to zero.

what it does
A templated daily page that auto-creates each session. Same prompts every day. The student fills in blanks, doesn't write from scratch.
how it applies, no chatbot
The templated bullets become wikilinks. *"Got stuck on"* links to skill cards. *"Asked"* links to peers. *"Read"* links to teacher-curated resources. The reflection itself becomes the navigation surface — every word a door to a human-authored artefact.

what it does
Each pursuit has a name, a description, prerequisites, progress, and a reward. The student chose it, so it doesn't feel assigned. Hints live inside the description, not in a chat.
how it applies, no chatbot
This is the most direct fit. Stones already have this shape. The win is the *prerequisites* line — every Stone links to the skill cards that unlock it. Stuck? Re-read the prereqs. The hint is the structure, not a generated answer.
07
borrowed from · maker studio crit

Pinned exemplars & floor walks

Visible reference work. The most powerful hint system ever invented.

Clean enclosure with cable management
teacher · pinned
Maya's PIR mount — uses zip ties
peer · same milestone
Last year · Aisha's final build
archive · 2024 cohort
teacher floor walk · today
"Three of you are stuck on resistor selection. Pinned a worked example to the board — start there."
what it does
A real studio teaches by showing, not telling. Exemplars on the wall, peer work alongside, and the teacher walking the floor noticing patterns.
how it applies, no chatbot
The single best chatbot replacement. Teacher pins exemplars per milestone; peer work surfaces automatically when the student is in the same phase. The floor walk note is a class-level broadcast, not a 1:1 generated message. Trust source: human, every time.

what it does
A visible path of nodes shows where you've been, where you are, and what's ahead. Locked nodes signal what's still to come without nagging.
how it applies, no chatbot
Take the *path*. Drop the streaks, hearts, and notification guilt. The "you are here" card is the perfect non-chatbot hint surface — fixed resources for the current node, visible to everyone, authored by the teacher.
